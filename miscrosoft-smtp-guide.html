Deployment Infrastructure Map

This sequential guide configures your custom domain infrastructure inside Microsoft 365. Completing these phases ensures authentication tokens are accepted by your Cloudflare Worker script over port 587 without tripping security firewalls or throwing runtime socket faults.

01

Domain Alignment & DKIM/MX Signing

Authorize your custom domain to send mail on behalf of the Microsoft tenant infrastructure.

Before initializing transaction channels, your root or subdomain records must point explicitly to Microsoft's mail exchange endpoints inside your Cloudflare DNS zone dashboard.

MX Record: web-asweq-com.mail.protection.outlook.com

To enable cryptographic DKIM signing keys:

  1. Go directly to the Microsoft Defender Protection Dashboard: security.microsoft.com/dkimv2
  2. Select your custom domain (web.asweq.com).
  3. Click Create DKIM keys. Microsoft will provision two unique CNAME records.
  4. Add these CNAME records directly into your Cloudflare DNS zone, then click Enable back in Microsoft.
02

Domain-Based Mailbox Creation

Provision a domain-mapped mailbox instance inside your active enterprise workspace tenant.

  1. Open the Microsoft 365 Active Users Panel: admin.cloud.microsoft/#/users
  2. Click Add a user.
  3. Under the domain identification inputs, assign your preferred prefix identifier name (info) and select your customized domain extension from the context dropdown (@web.asweq.com).
  4. Assign the required product license allocation checks (e.g., Business Basic) to instantiate a physical Exchange mailbox node container.
03

Manage Email Apps: Activate SMTP AUTH

Explicitly unlock protocol connectivity flags for the designated mail node container.

Microsoft disables legacy application app layer access sequences universally across new users. This must be bypassed explicitly on the user record:

  1. Open your Active Users List: admin.cloud.microsoft/#/users
  2. Click directly on the newly created info@web.asweq.com line to slide out the panel.
  3. On the configuration canvas that emerges, click on the center Mail panel tab.
  4. Under the Email apps section title, click Manage email apps.
  5. Scroll down, ensure the checkbox tracking Authenticated SMTP is marked Checked (True).
  6. Click Save changes.
04

Deactivate Global Security Defaults

Unlock the tenant-wide default filters that drop legacy authentication handshakes.

Modern tenants execute a hidden rule mechanism called Security Defaults that intercepts script requests and forces interactive multi-factor checks. To clear this via your portal settings:

  1. Open the Microsoft Entra Properties Dashboard: entra.microsoft.com (Tenant Properties)
  2. Scroll all the way down to the bottom of the properties page.
  3. Click the text link layout option at the base of the section: Manage security defaults.
  4. Flip the toggle switch parameters cleanly to Disabled. Mark any reason checkbox and click Save.
05

Elevate Custom Domain Address as Primary UPN Identity

Map your master login credential tracking string explicitly to your custom domain suffix footprint.

If you connect to port 587 using info@web.asweq.com while your tenant profile lists your primary login identifier string as web@asweq.onmicrosoft.com, Microsoft will drop the connection request with an authentication error.

  1. Open the Microsoft 365 Active Users Center: admin.cloud.microsoft/#/users
  2. Open your user profile line card, and under the Account tab click Manage username and email.
  3. Click the dropdown next to your user ID text field string, choose your verified domain (web.asweq.com), and adjust your string prefix cleanly to match your custom design identity.
  4. Alternatively, find info@web.asweq.com under the aliases array stack, click the three dots (...), and choose Set as primary email.
  5. Click Save changes.

Propagation Hold Notice: Microsoft takes up to 15–30 minutes to push identity credential shifts down to legacy Exchange SMTP edge nodes globally. Allow this time window to elapse completely before launching your production automation loops inside BarmgaMailer.
